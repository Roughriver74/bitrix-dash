# Деплой на Coolify

Инструкция по деплою Bitrix Dashboard на Coolify.

## Подготовка

1. Убедитесь, что у вас есть доступ к Coolify серверу
2. Подготовьте переменные окружения (см. ниже)

## Переменные окружения

В настройках приложения в Coolify добавьте следующие переменные:

### Обязательные:

- `BITRIX_WEBHOOK_URL` - URL вебхука Битрикс24 (формат: `https://your-domain.bitrix24.ru/rest/ID/token/`)
- `BITRIX_DEPARTMENT_NAME` - Название отдела для мониторинга (например: `IT отдел`)

### Опциональные:

- `UPSTASH_REDIS_REST_URL` - URL для Redis кеша (если используется)
- `UPSTASH_REDIS_REST_TOKEN` - Токен для Redis кеша
- `PORT` - Порт приложения (по умолчанию: 3000)
- `NODE_ENV` - Режим работы (production)

## Создание приложения в Coolify

1. В Coolify создайте новое приложение
2. Выберите тип: **Dockerfile**
3. Укажите путь к репозиторию или загрузите код
4. Убедитесь, что Dockerfile находится в корне проекта
5. Добавьте переменные окружения (см. выше)
6. Настройте домен и SSL (если нужно)

## Настройка Dockerfile

Проект уже содержит оптимизированный Dockerfile с multi-stage build:
- Stage 1: Установка зависимостей
- Stage 2: Сборка приложения
- Stage 3: Production образ с минимальным размером

## Проверка деплоя

После деплоя проверьте:

1. Откройте `/setup` для первоначальной настройки (если переменные не заданы)
2. Откройте `/dashboard` для просмотра дашборда
3. Откройте `/tasks` для управления задачами

## Troubleshooting

### Приложение не запускается

- Проверьте логи в Coolify
- Убедитесь, что все переменные окружения установлены
- Проверьте, что порт 3000 доступен

### Ошибки подключения к Bitrix24

- Проверьте правильность `BITRIX_WEBHOOK_URL`
- Убедитесь, что вебхук имеет необходимые права доступа
- Проверьте, что URL заканчивается на `/`

### Проблемы с кешем

- Если используется Redis, проверьте подключение
- Без Redis приложение будет работать, но без кеширования

## Обновление приложения

Для обновления приложения в Coolify:

1. Запустите новый деплой через интерфейс Coolify
2. Или настройте автоматический деплой при push в репозиторий

## Производительность

- Приложение использует standalone режим Next.js для оптимизации
- Рекомендуется минимум 512MB RAM
- Для production рекомендуется 1GB+ RAM
